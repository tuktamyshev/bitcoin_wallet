# Bitcoin HD-кошелёк — Лаборатория

Десктопное приложение-эмулятор иерархически-детерминированного (HD) Bitcoin-кошелька. Реализует полный цикл: от генерации мнемоники до подписания транзакций — с подробным журналом каждого криптографического шага.

> Приложение работает полностью офлайн. Блокчейн эмулируется встроенным in-memory ledger — реальные Bitcoin не используются.

---

pip install -r requirements.txt
python3 main.py

## Функционал

### 1. Генерация HD-кошелька (BIP39 / BIP32)

- Генерация мнемонической фразы на **12 или 24 слова** из криптографически стойкой энтропии (`os.urandom`).
- Вычисление **seed** из мнемоники через PBKDF2-HMAC-SHA512 (2048 итераций) с опциональной BIP39-фразой (passphrase).
- Деривация **мастер-ключа** по BIP32: `HMAC-SHA512(key="Bitcoin seed", data=seed) → 256-бит приватный ключ + 256-бит chain code`.
- Отображение **отпечатка seed** — первые 64 бита SHA-256 от seed; используется как короткий идентификатор для сравнения кошельков.

### 2. Деривация адресов по трём стандартам

Для каждого стандарта генерируется настраиваемое количество адресов (по умолчанию 5) для выбранного аккаунта:

| Стандарт | Путь деривации | Тип адреса | Префикс |
|----------|---------------|------------|---------|
| **BIP44** | `m/44'/0'/{account}'/0/{i}` | P2PKH (Legacy) | `1…` |
| **BIP49** | `m/49'/0'/{account}'/0/{i}` | P2SH-P2WPKH (Nested SegWit) | `3…` |
| **BIP84** | `m/84'/0'/{account}'/0/{i}` | P2WPKH (Native SegWit) | `bc1…` |

- Hardened-деривация (`'`) для purpose, coin type и account.
- Сжатие публичного ключа (33 байта: `02`/`03` + X-координата).
- Генерация адресов: SHA-256 → RIPEMD-160 → Base58Check (BIP44/49) или bech32 (BIP84).

### 3. Восстановление кошелька

- Ввод существующей мнемонической фразы в текстовое поле.
- Проверка контрольной суммы BIP39.
- Полная повторная деривация seed → мастер-ключ → адреса из той же фразы.
- Детерминированность: одна мнемоника всегда даёт одни и те же адреса и ключи.

### 4. Эмуляция транзакций

#### Пополнение (Coinbase)

- Эмулирует создание средств «из воздуха» (аналог награды за блок).
- Без входов (inputs), один выход (output) — целевой адрес.
- Комиссия: 0.

#### Перевод (ECDSA)

Полный цикл подписания транзакции:

1. **Формирование транзакции**: определение типа адреса отправителя, расчёт размера в vbytes, расчёт комиссии, формирование inputs/outputs (включая сдачу).
2. **ECDSA-подпись** (secp256k1): SHA-256 хеш данных транзакции → детерминированная подпись (r, s) по RFC 6979.
3. **Верификация**: проверка подписи публичным ключом.
4. **Обновление балансов**: списание с отправителя (сумма + комиссия), зачисление получателю, сдача обратно отправителю.

#### Комиссии

Эмуляция реалистичных комиссий в зависимости от типа адреса отправителя:

| Тип | Размер транзакции | Пример при 10 sat/vbyte |
|-----|------------------|------------------------|
| P2PKH (Legacy) | ~226 vbyte | 2260 sat |
| P2SH-P2WPKH (Nested SegWit) | ~167 vbyte | 1670 sat |
| P2WPKH (Native SegWit) | ~141 vbyte | 1410 sat |

- Настраиваемая ставка комиссии (sat/vbyte) в интерфейсе.
- Формула: `комиссия = размер_vbyte × ставка_sat/vbyte`.

#### txid

Каждая транзакция получает уникальный идентификатор — `double-SHA256` от JSON-представления транзакции (аналогично Bitcoin).

### 5. Валидация адресов

Перед отправкой адрес получателя проверяется:

- **Base58Check** (P2PKH, P2SH): декодирование, проверка контрольной суммы и байта версии (`0x00` для `1…`, `0x05` для `3…`).
- **bech32** (P2WPKH): проверка HRP (`bc`) и целостности данных.

### 6. Просмотр балансов

- Баланс каждого адреса отображается в таблице адресов и в выпадающих списках.
- **Агрегированный баланс**: общий баланс кошелька и баланс по каждому стандарту (BIP44/49/84) на вкладке «Обзор».
- Автоматическое обновление при любой операции через систему listener-ов в ledger.

### 7. Два кошелька

- Приложение поддерживает **два независимых кошелька** (A и B), работающих с общим ledger.
- Переключение между кошельками кнопками «Кошелёк A» / «Кошелёк B» в верхней панели.
- Кнопка **«Разделить экран»** — отображение обоих кошельков бок о бок (split view) для удобного перевода между ними.
- Общий ledger позволяет отправлять средства с адресов кошелька A на адреса кошелька B и наоборот.

### 8. Журнал (постоянная панель)

Нижняя панель каждого кошелька — подробный журнал всех операций:

- Пошаговое описание генерации кошелька (энтропия → мнемоника → seed → мастер-ключ → адреса).
- Детальное логирование каждой транзакции: inputs, outputs, размер, комиссия, подпись, верификация, txid.
- Объяснение криптографических операций (BIP39, BIP32, ECDSA, SHA-256, RIPEMD-160).

### 9. Приватные ключи

- По умолчанию скрыты.
- Кнопка «Приватные ключи» / «Скрыть ключи» на вкладке «Адреса» для переключения видимости.
- Двойной клик по строке таблицы — копирование адреса в буфер обмена.

### 10. JSON-инспектор

Вкладка «JSON» — полный дамп данных кошелька в реальном времени:

- Мнемоника, seed, отпечаток, параметры деривации.
- Все адреса с приватными ключами.
- Текущие балансы каждого адреса.
- Полная история транзакций.

## Используемые алгоритмы и стандарты

| Алгоритм / Стандарт | Где используется |
|---------------------|-----------------|
| **BIP39** | Генерация мнемоники, деривация seed (PBKDF2-HMAC-SHA512) |
| **BIP32** | Иерархическая деривация ключей (HMAC-SHA512, secp256k1) |
| **BIP44 / BIP49 / BIP84** | Пути деривации для разных типов адресов |
| **ECDSA (secp256k1)** | Подпись и верификация транзакций |
| **RFC 6979** | Детерминированная генерация nonce для ECDSA |
| **SHA-256** | Хеширование (адреса, txid, отпечаток seed) |
| **RIPEMD-160** | Хеширование публичного ключа для адресов |
| **HMAC-SHA512** | BIP32 деривация (мастер-ключ, дочерние ключи) |
| **PBKDF2-HMAC-SHA512** | BIP39 деривация seed из мнемоники |
| **Base58Check** | Кодирование Legacy (P2PKH) и Nested SegWit (P2SH) адресов |
| **bech32** | Кодирование Native SegWit (P2WPKH) адресов |

## Интерфейс

Тёмная тема в стиле GitHub Dark. Основные области:

- **Левая панель** — настройки кошелька: выбор длины фразы, количества адресов, аккаунта, BIP39-passphrase, кнопки создания/восстановления, поле мнемоники.
- **Правая панель** — вкладки: Обзор, Адреса (BIP44/49/84), Транзакции (пополнение + перевод + история), JSON.
- **Нижняя панель** — журнал с подробным описанием всех криптографических операций.
- **Верхний тулбар** — переключение кошельков A/B и кнопка разделения экрана.
